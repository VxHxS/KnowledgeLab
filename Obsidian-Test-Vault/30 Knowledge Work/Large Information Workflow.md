# Large Information Workflow

## Intake

Collect Markdown notes, PDFs, documentation exports, transcripts, manuals, and project logs. Keep original files when possible and write short summary notes that link to them.

## Processing

Use Obsidian for human-readable organization and LightRAG for retrieval over the same material.

Recommended note fields:

- Source
- Topic
- Status
- Key claims
- Open questions
- Related projects

## Query examples

- What Unity profiling issues have appeared in my notes?
- What mastering decisions did I make for the last version?
- Which sources mention LUFS, true peak, or limiter oversampling?
- What are the unresolved questions in the current project?
