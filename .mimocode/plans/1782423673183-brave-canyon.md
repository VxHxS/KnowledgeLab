## Проблема 1: Анимация — бегущая строка работает неправильно

### Текущее состояние
- `_frame_ms = 60` — слишком быстро
- 3 точки (`num_dots = 3`) бегают по периметру
- Палитра из 4 цветов — мало разнообразия
- Рамка `#e0e7ee` — тусклая

### Решение

**Файл: `scripts/knowledgelab/ui/message_bubble.py`**

1. **Скорость**: `_frame_ms = 60` → `_frame_ms = 120` (в 2 раза медленнее)
2. **Фаза**: `_phase += 0.04` → `_phase += 0.02` (медленнее движение)
3. **Точки → линии**: вместо `create_oval` рисовать `create_line` с градиентом — несколько тонких сегментов которые появляются/исчезают
4. **Палитра**: добавить больше цветов — яркие неоновые оттенки:
   ```python
   palette=("#3b82f6", "#8b5cf6", "#06b6d4", "#10b981", "#f59e0b", "#ef4444", "#ec4899")
   ```
5. **Рамка**: `#e0e7ee` → `#d1d5db` (чуть ярче, но всё ещё тонкая 1px)
6. **Количество линий**: `num_dots = 3` → `num_lines = 4` (4 сегмента)
7. **Анимация ТОЛЬКО на thinking-пузыре**: в `start_thinking_animation()` вызывать `start_animation()`, в `_stop_thinking_animation()` — `stop_animation()`. Ответы и пользовательские сообщения — без анимации.

## Проблема 2: Ярлык не всегда ссылается на последнюю версию

### Текущее состояние
- `C:\Users\Юрий\Desktop\LightRag\LightRAG-Chat.lnk` указывает на `KnowledgeLab-staging-20260605211949`
- Если создать новую папку staging, ярлык устареет

### Решение

Создать **обёртку** в `C:\Users\Юрий\Desktop\LightRag\` — PowerShell-скрипт `Launch-KnowledgeLab.ps1`, который:
1. Ищет последнюю папку `KnowledgeLab-staging-*` в `Documents\Freelance\`
2. Запускает `LightRAG-Desktop-Chat.ps1` из найденной папки
3. Ярлык `.lnk` указывает на этот скрипт-обёртку

**Новый файл: `desktop-launchers/Launch-KnowledgeLab.ps1`**
```powershell
$freelance = Join-Path $env:USERPROFILE "Documents\Freelance"
$latest = Get-ChildItem -Path $freelance -Directory -Filter "KnowledgeLab-staging-*" |
    Sort-Object Name -Descending | Select-Object -First 1
if ($latest) {
    $script = Join-Path $latest.FullName "LightRAG-Desktop\LightRAG-Desktop-Chat.ps1"
    & $script
} else {
    Write-Host "KnowledgeLab not found"
}
```

**Обновить ярлык**: создать `.lnk` → `Launch-KnowledgeLab.ps1`

## Проблема 3: Drag-and-drop не работает

### Текущее состояние
- `tkinterdnd2` **не установлен** в Python окружении
- DnD код в `main.py` пытается импортировать → ловит ImportError → отключает DnD

### Решение

1. **Установить tkinterdnd2** в venv:
   ```
   pip install tkinterdnd2
   ```

2. **Если venv нет** — установить в системный Python или создать venv:
   ```
   python -m venv LightRAG\.venv
   LightRAG\.venv\Scripts\pip install -r requirements-all.txt
   ```

3. **В requirements-core.txt** добавить `tkinterdnd2` (если ещё нет)
